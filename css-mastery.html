บทที่ 1-5 CSS Fundamentals, Selectors, Units & Colors

Cascade & Specificity (คะแนนความสำคัญ):

  • แท็กทั่วไป p = 1 คะแนน
  • Class .btn = 10 คะแนน
  • ID #header = 100 คะแนน
  • Inline style style="..." = 1000 คะแนน
  • !important = ชนะทุกอย่าง (แต่ห้ามใช้พร่ำเพรื่อ)

Units (หน่วยวัด):

  • px: พิกเซลตายตัว — ไม่ขยายตามการตั้งค่า font ผู้ใช้
  • rem: ขยายตาม root font-size (16px ปกติ) — ดีสำหรับ Accessibility
  • em: ขยายตาม font-size ของ parent — ใช้สำหรับ spacing ที่ scale ตาม text
  • vw / vh: เปอร์เซ็นต์ตามความกว้าง/สูงของ viewport
  • %: เปอร์เซ็นต์ตาม parent element

Colors: hex (#ffffff), rgb(), rgba() (มีความโปร่งใส alpha), hsl() (Hue, Saturation, Lightness — เหมาะสร้าง color system)

⚠️ ข้อควรระวัง: หลีกเลี่ยง !important — มันทำลายระบบ Specificity และทำให้โค้ดแก้ไขได้ยากมากในอนาคต ถ้าต้องใช้ !important แปลว่าโครงสร้าง CSS มีปัญหาอยู่ก่อนแล้ว

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: #id p.active มีคะแนน Specificity เท่าไหร่?

✅ 111 คะแนน — #id (100) + p (1) + .active (10) = 111

Q2: ทำไม rem ดีกว่า px สำหรับ font-size?

✅ rem ขยายตาม root font-size ของเบราว์เซอร์ ทำให้ผู้ใช้ที่ตั้งค่าตัวอักษรใหญ่ขึ้น (เพราะสายตาไม่ดี) ยังใช้เว็บได้ ส่วน px ตายตัวไม่สนใจ setting ผู้ใช้

Q3: rgba(0, 0, 0, 0.5) หมายถึงสีอะไร?

✅ สีดำ (0,0,0) ที่โปร่งแสง 50% — ค่า alpha 0=โปร่งใสสมบูรณ์, 1=ทึบสมบูรณ์

บทที่ 6-8 Box Model, Display & Position

Box Model: ทุก element คือกล่องสี่เหลี่ยม ประกอบด้วย (จากในสุดออกนอก): Content → Padding → Border → Margin

Display values สำคัญ:

  • block: กินพื้นที่เต็มบรรทัด ขึ้นบรรทัดใหม่เสมอ (div, p, h1)
  • inline: เรียงแนวนอน ปรับ width/height ไม่ได้ (span, a, strong)
  • inline-block: เรียงแนวนอน + ปรับขนาดได้
  • none: ซ่อน element ออกจาก layout ทั้งหมด

Position values:

  • static: default — ไหลตาม document flow ปกติ
  • relative: ขยับจากตำแหน่งเดิมของตัวเอง ยังกินพื้นที่เดิม
  • absolute: หลุดจาก flow ไปอ้างอิงกับ parent ที่ไม่ใช่ static
  • fixed: ติดกับ viewport ไม่เลื่อนตามหน้า
  • sticky: ไหลปกติจนถึงจุดที่กำหนด แล้วค้างไว้
⚠️ ข้อควรระวัง: โดยเริ่มต้นเบราว์เซอร์ใช้ box-sizing: content-box — ถ้าตั้งความกว้าง 100px แล้วใส่ padding 20px กล่องจะขยายเป็น 140px! แก้ไขด้วย: ใส่ *, *::before, *::after { box-sizing: border-box; } ไว้บนสุดของ CSS ทุกโปรเจกต์

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ความแตกต่างระหว่าง display:none และ visibility:hidden คืออะไร?

✅ display:none ลบ element ออกจาก layout ทั้งหมด (ไม่กินพื้นที่) ส่วน visibility:hidden ซ่อนแต่ยังกินพื้นที่เดิมอยู่ (มีช่องว่างโล่งๆ)

Q2: Margin Collapse คืออะไร?

✅ เมื่อ block element 2 ตัวอยู่ติดกันในแนวตั้ง margin จะ "พับรวมกัน" เหลือแค่ค่ามากสุด เช่น margin-bottom: 20px + margin-top: 30px = ระยะห่างแค่ 30px (ไม่ใช่ 50px) ปัญหานี้ไม่เกิดใน Flex/Grid

Q3: position: sticky ต่างจาก fixed อย่างไร?

✅ sticky ไหลตาม document flow ปกติก่อน แล้วค้างอยู่กับที่เมื่อเลื่อนถึงจุดที่กำหนด (เช่น header ที่ค้างเมื่อ scroll) ส่วน fixed ออกจาก flow ตลอด ติดกับ viewport ทันที

บทที่ 9-11 Flexbox, Grid Layout & Responsive Design

Flexbox (1 มิติ): จัดเรียง items ในแนวแถว (row) หรือแนวคอลัมน์ (column) ทีละแกน

  • display: flex: เปิดใช้ Flexbox บน container
  • justify-content: จัดเรียงตาม Main Axis (ค่าเริ่มต้นคือแนวนอน)
  • align-items: จัดเรียงตาม Cross Axis
  • flex-wrap: wrap: อนุญาตให้ item ขึ้นบรรทัดใหม่
  • gap: ระยะห่างระหว่าง items
  • flex: 1: item นี้ขยายเต็มพื้นที่ที่เหลือ

Grid (2 มิติ): จัดเรียงทั้ง row และ column พร้อมกัน

  • grid-template-columns: repeat(3, 1fr): 3 คอลัมน์กว้างเท่ากัน
  • grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)): Responsive Grid อัตโนมัติ
  • grid-column: 1 / 3: item กินพื้นที่ข้ามคอลัมน์

Responsive Design: @media (max-width: 768px) { ... } — Mobile First: เขียน style มือถือก่อน แล้ว override ด้วย min-width

⚠️ ข้อควรระวัง: เมื่อใช้ flex-direction: column แกนจะสลับ! justify-content จะจัดแนวตั้งแทน และ align-items จะจัดแนวนอนแทน ทำให้โค้ดที่เคย center ในแนวนอนกลับไม่ทำงานเมื่อเปลี่ยน direction

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: เมื่อไหรควรใช้ Flexbox และเมื่อไหรควรใช้ Grid?

✅ ใช้ Flexbox เมื่อจัดเรียงใน 1 ทิศทาง (เช่น navbar, button group, card items ในแถว) ใช้ Grid เมื่อต้องการ layout 2 มิติ (เช่น หน้า dashboard, magazine layout, photo gallery)

Q2: 1fr หมายความว่าอะไร?

✅ "1 fraction" หรือ 1 ส่วนของพื้นที่ว่างที่เหลือ ถ้า grid-template-columns: 1fr 2fr 1fr = แบ่งเป็น 4 ส่วน คอลัมน์ 1 ได้ 1 ส่วน คอลัมน์ 2 ได้ 2 ส่วน คอลัมน์ 3 ได้ 1 ส่วน

Q3: Mobile First กับ Desktop First ต่างกันอย่างไร และอันไหนดีกว่า?

✅ Mobile First: เขียน CSS สำหรับมือถือก่อน ใช้ min-width ขยายออก ส่วน Desktop First: เขียนสำหรับ desktop แล้วใช้ max-width ลดลง Mobile First ดีกว่าเพราะ CSS โหลดน้อยกว่าบนมือถือ และเป็นมาตรฐานของ Tailwind, Bootstrap ยุคใหม่

บทที่ 12-14 Transform, Transition & Animation

Transform: เปลี่ยนรูปทรงโดยไม่กระทบ layout ของ element อื่น

  • translate(x, y): ขยับตำแหน่ง
  • rotate(45deg): หมุน
  • scale(1.2): ขยาย/ย่อ
  • skew(10deg): บิดเฉียง
  • สามารถรวมกันได้: transform: translateX(10px) rotate(45deg) scale(1.1)

Transition: ทำให้การเปลี่ยนสไตล์เกิดขึ้นอย่างนุ่มนวล

  • transition: property duration easing delay
  • ตัวอย่าง: transition: all 0.3s ease-in-out

Animation: ควบคุมความเคลื่อนไหวเป็น keyframe

  • @keyframes name { from { } to { } }
  • animation: name duration easing iteration
  • animation-fill-mode: forwards: หยุดที่ keyframe สุดท้าย
⚠️ ข้อควรระวัง: อย่า animate ด้วย width, height, top, margin — คุณสมบัติเหล่านี้ทำให้เบราว์เซอร์คำนวณ Layout ใหม่ทั้งหน้าในทุก frame (Layout Reflow) ทำให้กระตุกมาก ให้ใช้ transform และ opacity แทน เพราะ GPU จัดการได้โดยตรง

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทำไม transform: translate() ดีกว่าการเปลี่ยน top/left สำหรับ animation?

✅ transform ประมวลผลด้วย GPU (Compositor Thread) โดยตรง ไม่ต้องให้ CPU คำนวณ Layout ใหม่ ทำให้ได้ 60fps ลื่นไหล ส่วน top/left trigger Layout → Paint → Composite ทุก frame ทำให้กระตุก

Q2: animation-iteration-count: infinite กับ 3 ต่างกันอย่างไร?

✅ infinite วนซ้ำไม่หยุด ส่วน 3 เล่น animation 3 รอบแล้วหยุด ถ้าต้องการหยุดที่ state สุดท้ายให้เพิ่ม animation-fill-mode: forwards

บทที่ 15-17 Pseudo Classes, Pseudo Elements & CSS Variables

Pseudo Classes (สถานะ): ตรวจจับสถานะของ element:

  • :hover: เมื่อเมาส์ชี้
  • :focus: เมื่อกำลัง focus (tab หรือคลิก)
  • :active: เมื่อกำลังคลิกค้างอยู่
  • :nth-child(2n): เลือกตัวที่ 2, 4, 6... (เลขคู่)
  • :not(.excluded): เลือกทุกตัวที่ไม่ตรง selector ที่กำหนด
  • :has(img): เลือก element ที่มี img ข้างใน (CSS ยุคใหม่)

Pseudo Elements (สร้าง element จำลอง):

  • ::before / ::after: เสก element จำลองโดยไม่เพิ่ม HTML — ต้องมี content: "" เสมอ
  • ::first-letter: ตัวอักษรแรก (เช่น Drop Cap)
  • ::placeholder: สไตล์ข้อความ placeholder ใน input
  • ::selection: สไตล์เมื่อผู้ใช้ drag select ข้อความ

CSS Variables (Custom Properties): ประกาศตัวแปรด้วย --name: value แล้วใช้ด้วย var(--name)

  • ประกาศที่ :root เพื่อใช้ได้ทั้งไฟล์
  • เปลี่ยนค่าด้วย JS: document.documentElement.style.setProperty('--color', 'red')
⚠️ ข้อควรระวัง: ::before และ ::after จะไม่แสดงผลถ้าลืม content: "" — แม้แต่ค่าว่างก็ต้องใส่ นอกจากนี้ Pseudo elements ใช้ :: (2 colons) แต่ Pseudo classes ใช้ : (1 colon)

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทำไม ::before ถึงไม่แสดงผลทั้งที่เขียน CSS ถูกต้อง?

✅ 99% ของกรณีคือลืมใส่ content: ""; — แม้แต่ค่าว่างก็ต้องมี เพราะนี่คือ required property ของ pseudo elements

Q2: CSS Variables ต่างจาก SASS Variables อย่างไร?

✅ CSS Variables (Custom Properties) ทำงาน runtime ใน browser สามารถเปลี่ยนค่าด้วย JavaScript ได้ และรู้จัก CSS Cascade ส่วน SASS Variables ทำงานตอน compile time เป็นแค่ shortcut การเขียนโค้ด ไม่สามารถ dynamic ได้ใน browser

บทที่ 18-19 Advanced & Modern CSS Features

CSS Architecture:

  • BEM (Block__Element--Modifier): ระบบตั้งชื่อ class เช่น .card__title--highlighted — Block=card, Element=title, Modifier=highlighted ป้องกัน class ชนกัน
  • SMACSS: แบ่ง CSS เป็น Base, Layout, Module, State, Theme
  • Utility-first (Tailwind): ใช้ class สำเร็จรูปสั้นๆ เช่น flex gap-4 p-6 text-pink-500

Modern CSS Functions:

  • clamp(min, preferred, max): font-size ที่ scale ตามหน้าจออัตโนมัติ เช่น font-size: clamp(1rem, 2.5vw, 2rem)
  • calc(100% - 40px): คำนวณขนาดผสมหน่วยต่างกัน
  • min() / max(): เลือกค่าที่เล็ก/ใหญ่ที่สุด

Modern CSS Spec (ปี 2024+):

  • CSS Nesting: เขียน CSS ซ้อนกันได้โดยตรงเหมือน SCSS แล้ว (browser รองรับแล้ว)
  • Container Queries: @container — สไตล์ตามขนาดของ parent container (ไม่ใช่ viewport) ยืดหยุ่นกว่า Media Query มาก
  • CSS Layers: @layer — จัดลำดับความสำคัญของ CSS ชัดเจนขึ้น
⚠️ ข้อควรระวัง: BEM ทำให้ชื่อ class ยาวมาก ถ้าใช้กับ framework อย่าง React ที่มี CSS Modules หรือ Tailwind อยู่แล้ว อาจไม่จำเป็น ให้เลือก methodology ตามขนาดทีมและ stack ที่ใช้

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: clamp(1rem, 2vw, 3rem) จะทำงานอย่างไรบนหน้าจอกว้าง 1440px?

✅ 2vw × 1440px = 28.8px ≈ 1.8rem — อยู่ระหว่าง min (1rem=16px) และ max (3rem=48px) จึงใช้ 1.8rem บนหน้าจอเล็กกว่า min-width ที่ให้ 2vw < 1rem จะใช้ 1rem แทน

Q2: Container Queries ดีกว่า Media Queries อย่างไร?

✅ Media Queries ตรวจขนาด viewport (หน้าจอทั้งหมด) ส่วน Container Queries ตรวจขนาดของ parent container ทำให้ component ปรับสไตล์ตามพื้นที่จริงที่ตัวเองมี เหมาะกับ reusable components ที่ใช้ได้ทั้ง sidebar (แคบ) และ main (กว้าง)
💡 Pro Tip — เส้นทาง CSS ต่อไป: หลังจากเชี่ยวชาญ CSS แล้ว ให้เรียน Tailwind CSS (Utility-first) เพราะเป็น standard ในอุตสาหกรรมในปัจจุบัน จากนั้นเรียน CSS-in-JS (Styled Components / Emotion) สำหรับ React projects และ CSS Modules สำหรับ Next.js
