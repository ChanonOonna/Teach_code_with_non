บทที่ 1 HTML คืออะไร & ประวัติความเป็นมา

History: คิดค้นโดย Tim Berners-Lee ในปี 1991 เพื่อใช้แชร์ข้อมูลระหว่างนักวิจัย

HTML4: เน้นการแสดงผลเป็นหลัก แต่โค้ดมักจะกระจัดกระจาย ไร้ระเบียบ

XHTML: นำเอาโครงสร้างไวยากรณ์ที่เข้มงวดของ XML มาครอบทับ ถ้าลืมปิดแท็ก หน้าเว็บจะไม่แสดงผลทันที

HTML5: มาตรฐานปัจจุบันที่ยืดหยุ่น รองรับการเล่นวิดีโอ/เสียงในตัว และแก้ปัญหาโครงสร้างที่ยุ่งยากในอดีต

W3C vs WHATWG: เดิมที W3C คุมกฎเกณฑ์ แต่ทำงานช้า กลุ่มเบราว์เซอร์จึงตั้ง WHATWG ขึ้นมาเพื่อผลักดันมาตรฐาน "Living Standard" (อัปเดตตลอดเวลา) ซึ่งเป็นมาตรฐานหลักที่เราใช้ในปัจจุบัน

⚠️ ข้อควรระวัง: อย่าพยายามมองว่า HTML เป็น Programming Language เพราะมันไม่มี Logic หรือการคำนวณ มันคือ Markup Language เท่านั้น หากเขียนแท็กผิด เบราว์เซอร์จะไม่แจ้ง Error แต่จะพยายามเดาและแสดงผลแบบเพี้ยนๆ แทน

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: HTML ย่อมาจากอะไร?

✅ HyperText Markup Language — "HyperText" หมายถึงข้อความที่มีลิงก์เชื่อมถึงกันได้ "Markup" หมายถึงการใช้แท็กกำกับความหมายของเนื้อหา

Q2: HTML5 เพิ่มความสามารถสำคัญอะไรที่ HTML4 ไม่มี?

✅ เล่น Video/Audio ในตัวโดยไม่ต้องลง Flash Plugin, รองรับ Semantic tags (<header>, <nav> ฯลฯ), Canvas API, Local Storage, Geolocation

Q3: ถ้าเขียน HTML tag ผิดรูปแบบ จะเกิดอะไรขึ้น?

✅ เบราว์เซอร์จะไม่แสดง Error แต่จะพยายาม "เดา" และแก้ไขให้ด้วยตัวเอง (Browser Error Recovery) ซึ่งอาจทำให้หน้าเว็บแสดงผลไม่ตรงตามที่ต้องการ

บทที่ 2 HTML Document Structure

สิ่งที่จำเป็นต้องรู้ในโครงสร้างหลัก:

  • <!DOCTYPE html>: ไม่ใช่แท็ก HTML แต่เป็นคำสั่งบอกเบราว์เซอร์ว่า "นี่คือเอกสาร HTML5"
  • <html lang="th">: Root tag ที่ครอบทั้งหมด ระบุภาษาช่วย Screen Reader และ SEO
  • <head>: คลังข้อมูลจำเพาะ ห้ามใส่ <h1> หรือ <p> ในนี้เด็ดขาด
  • <body>: รวบรวมทุกมิติข้อมูลที่จะปรากฏแก่สายตาผู้ใช้งาน
⚠️ ข้อควรระวัง: ลืมเขียน <!DOCTYPE html> จะทำให้เบราว์เซอร์เข้าสู่ "Quirks Mode" ซึ่งจะส่งผลให้การคำนวณ Layout ของ CSS เพี้ยนและไม่แสดงผลตามสไตล์โมเดิร์น
ทดลองโครงสร้างพื้นฐาน

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ความแตกต่างระหว่าง <head> และ <body> คืออะไร?

✅ <head> เก็บ metadata ที่ผู้ใช้ไม่เห็น (title, meta, script, link) ส่วน <body> เก็บเนื้อหาที่แสดงบนหน้าจอทั้งหมด

Q2: ทำไมต้องใส่ lang="th" ในแท็ก <html>?

✅ ช่วย Screen Reader อ่านออกเสียงภาษาได้ถูกต้อง, ช่วย Browser เลือก Font ที่เหมาะสม, และช่วย SEO ให้ Google รู้ว่าเนื้อหาเป็นภาษาไทย

สร้างหน้าเว็บ HTML ที่สมบูรณ์มีหัวข้อ, ย่อหน้า 2 อัน, และ title ที่ถูกต้อง โดยไม่ดูตัวอย่างด้านบน (จำโครงสร้างให้ได้)

บทที่ 3 Metadata & Tags สำคัญ

แท็ก <meta> ในส่วน <head> มีผลอย่างยิ่งต่อพฤติกรรมของเว็บ:

  • charset="UTF-8": จัดการระบบตัวอักษรให้รองรับภาษาไทยและสัญลักษณ์พิเศษทั่วโลก
  • name="viewport" content="width=device-width, initial-scale=1.0": สั่งให้ขนาดหน้าเว็บยืดหยุ่นตามความกว้างของมือถือ
  • name="description": ชุดข้อความที่จะไปแสดงบนหน้าค้นหาของ Google (SERP)
  • name="robots" content="noindex": บอก Google ว่าอย่าเก็บหน้านี้เข้าฐานข้อมูลค้นหา
  • Open Graph: property="og:title", og:image" — ควบคุมหน้าตาเวลา share ลิงก์บน Social Media
⚠️ ข้อควรระวัง: การลืมใส่ viewport meta tag จะทำให้หน้าเว็บบนมือถือแสดงผลในสเกลย่อส่วนขนาดเท่าหน้าจอเดสก์ท็อป ตัวอักษรจะเล็กมากจนผู้ใช้งานต้องซูม

Q1: Open Graph tags ใช้ทำอะไร?

✅ ควบคุมหน้าตาเวลา Share ลิงก์บน Facebook, LINE, Twitter — กำหนดได้ว่าจะแสดงรูป thumbnail อะไร, หัวข้ออะไร, คำอธิบายอะไร

บทที่ 4 Text Elements เจาะลึกข้อความ

ทำความเข้าใจความต่างระหว่างการเลือกใช้แท็กเพื่อให้เหมาะสมต่อโครงสร้างข้อมูลและ SEO:

  • <h1> ถึง <h6>: ลำดับหัวข้อ ความสำคัญสูงสุดอยู่ที่ h1
  • <strong> vs <b>: ทั้งคู่แสดงผลตัวหนา แต่ strong สื่อความหมายว่า "สำคัญมาก" ต่อ Search Engine
  • <em> vs <i>: ทั้งคู่แสดงตัวเอียง แต่ em สื่อถึงการเน้นเสียง
  • <mark>: ไฮไลต์ข้อความ (สีเหลือง)
  • <del> / <ins>: แสดงการลบ/เพิ่มข้อมูล (เช่น ราคาเดิม ที่ขีดฆ่า)
  • <sub> / <sup>: ตัวห้อยล่าง / ตัวยกบน (H₂O, x²)
  • <blockquote>: คำพูดอ้างอิง
  • <span>: Inline container ไม่มีความหมาย ใช้จัดสไตล์เฉพาะจุด
⚠️ ข้อควรระวัง: ใน 1 หน้าเว็บ ควรมีแท็ก <h1> เพียงแค่อันเดียวเท่านั้น (ชื่อหัวข้อหลักของหน้า) การใส่หลายอันจะทำลายคะแนน SEO

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทำไม <strong> ดีกว่า <b> สำหรับ SEO?

✅ <strong> มี semantic meaning — บอก Search Engine ว่าข้อความนั้น "มีความสำคัญ" ส่วน <b> เปลี่ยนแค่รูปลักษณ์ภายนอก ไม่มีความหมายเพิ่ม

Q2: <span> ต่างจาก <div> อย่างไร?

✅ <span> เป็น inline element (ไม่ขึ้นบรรทัดใหม่) ส่วน <div> เป็น block element (กินพื้นที่เต็มบรรทัด ขึ้นบรรทัดใหม่เสมอ)

บทที่ 5 Lists & Tables การจัดการข้อมูลเชิงโครงสร้าง

Lists:

  • <ul>: Unordered List (จุดนำหน้า ไม่มีลำดับ)
  • <ol>: Ordered List (มีเลขลำดับ)
  • <dl>: Definition List — คู่ <dt> (คำศัพท์) และ <dd> (คำนิยาม)

Tables: โครงสร้างตารางที่สมบูรณ์ต้องแยกส่วนชัดเจน:

  • <thead>: ส่วนหัวตาราง
  • <tbody>: ส่วนเนื้อหา
  • <tfoot>: ส่วนสรุปท้ายตาราง
  • colspan / rowspan: รวม cell แนวนอน / แนวตั้ง
⚠️ ข้อควรระวัง: ภายใน <ul> หรือ <ol> ลูกโดยตรง (Direct Child) ต้องเป็น <li> เท่านั้น ห้ามนำ <div> หรือ <p> ไปใส่ครอบโดยตรง นอกจากนี้ อย่าใช้ Table สำหรับจัด Layout หน้าเว็บ — Table มีไว้สำหรับข้อมูลตารางเท่านั้น ใช้ Flexbox/Grid แทน

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: colspan="3" หมายความว่าอะไร?

✅ Cell นั้นจะขยายกินพื้นที่ 3 คอลัมน์ในแนวนอน มักใช้ใน <tfoot> สำหรับแถวสรุปที่ต้องรวม column เข้าหากัน

Q2: เมื่อไหรควรใช้ <ol> แทน <ul>?

✅ ใช้ <ol> เมื่อลำดับสำคัญ เช่น ขั้นตอนวิธีทำอาหาร, การ setup โปรแกรม, Ranking ใช้ <ul> เมื่อลำดับไม่สำคัญ เช่น รายการของชำ, รายการเมนู

บทที่ 6-7 Links, Images & Responsive Assets

Links (<a>):

  • href="#id": ลิงก์ไปยัง element ในหน้าเดียวกัน (Anchor Link)
  • target="_blank" rel="noopener noreferrer": เปิดแท็บใหม่ + ปิดช่องโหว่ความปลอดภัย
  • download: บังคับให้ดาวน์โหลดไฟล์แทนการเปิด

Images (<img>):

  • alt: คำอธิบายรูปภาพ (บังคับสำหรับ Accessibility)
  • loading="lazy": ชะลอโหลดรูปที่ยังเลื่อนไปไม่ถึง ช่วยเว็บโหลดเร็ว
  • srcset: ระบุรูปหลายขนาด ให้เบราว์เซอร์เลือกตามความละเอียดหน้าจอ

Responsive Images: ใช้ <picture> + <source> เพื่อเปลี่ยนไฟล์รูปตามขนาดหน้าจอ

⚠️ ข้อควรระวัง: 1. การใช้ target="_blank" ต้องพ่วง rel="noopener noreferrer" เสมอ เพื่อป้องกันการโจมตีแบบ "Reverse Tabnapping"
2. ห้ามลืมใส่ alt บนแท็กรูปภาพ หากรูปเป็นของตกแต่งล้วนๆ ให้ใส่ alt="" (ค่าว่าง) เพื่อบอก Screen Reader ว่าข้ามได้

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทำไม loading="lazy" ช่วยให้เว็บเร็วขึ้น?

✅ เบราว์เซอร์จะโหลดรูปเฉพาะที่อยู่ในหรือใกล้ Viewport เท่านั้น รูปด้านล่างที่ผู้ใช้ยังไม่เลื่อนถึงจะถูกข้ามไปก่อน ทำให้หน้าโหลดเร็วขึ้นและประหยัด bandwidth

Q2: ถ้ารูปภาพเป็นของตกแต่งล้วนๆ ควรใส่ alt อย่างไร?

✅ ใส่ alt="" (ค่าว่าง) ไม่ใช่ลบ alt ทิ้ง — เพื่อบอก Screen Reader ว่ารูปนี้ไม่มีข้อมูลที่ต้องอ่าน ถ้าลบ alt ทิ้ง Screen Reader จะอ่าน URL ของรูปแทน ซึ่งฟังดูแย่มาก

บทที่ 9-11 Forms, Input Types & Validation

การจัดการแบบฟอร์มเพื่อรวบรวมข้อมูล โดยมี HTML คอยตรวจเช็กความถูกต้องเบื้องต้น (Client-side Validation):

  • required: บังคับว่าช่องนี้ห้ามเว้นว่าง
  • pattern="[A-Za-z]{3}": ใช้ Regex จำกัดรูปแบบการกรอก
  • min / max: กำหนดช่วงค่าตัวเลขหรือวันที่
  • minlength / maxlength: จำกัดความยาวข้อความ
  • type="hidden": อินพุตซ่อนค่า มักใช้ส่ง Token ความปลอดภัย
  • <label for="id">: เชื่อม label กับ input — คลิก label แล้ว input ได้รับ focus

Input Types สำคัญ: text, email, password, number, date, file, checkbox, radio, range, color

⚠️ ข้อควรระวัง: HTML Validation เป็นเพียงด่านป้องกันขั้นแรกเท่านั้น ผู้ใช้งานที่รู้ไอทีสามารถลบ required ออกได้ง่ายด้วย DevTools ดังนั้นจำเป็นต้องมี Backend Validation ที่เซิร์ฟเวอร์ด้วยเสมอ

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทำไมต้องใช้ <label for="id"> แทนการพิมพ์ข้อความธรรมดา?

✅ Label ที่เชื่อมกับ input (ผ่าน for/id) จะทำให้ (1) คลิก label แล้ว input ได้รับ focus, (2) Screen Reader อ่านออกเสียงชื่อ field ให้ผู้พิการทางสายตา, (3) พื้นที่คลิกใหญ่ขึ้นบนมือถือ

Q2: type="email" ต่างจาก type="text" อย่างไร?

✅ type="email" จะตรวจสอบว่าต้องมี @ และ domain ในรูปแบบที่ถูกต้อง, บนมือถือจะแสดง keyboard ที่มีปุ่ม @ โดยอัตโนมัติ, และ autocomplete แนะนำ email ที่เคยกรอก

💪 Challenge

สร้างฟอร์มลงทะเบียน มีฟิลด์: ชื่อ (required, minlength 2), อีเมล (email, required), รหัสผ่าน (password, minlength 8), อายุ (number, min 13), และปุ่ม submit

บทที่ 12-15 Semantic HTML, Multimedia & Drawing

Semantic HTML: เปลี่ยนจากบล็อกไร้ความหมาย (div) มาเป็นแท็กที่ระบุหน้าที่ในตัวเองชัดเจน:

  • <header>: ส่วนหัว (โลโก้, ชื่อเว็บ)
  • <nav>: เมนูนำทาง
  • <main>: เนื้อหาหลักของหน้า (มีได้แค่อันเดียว)
  • <article>: บทความที่เป็นอิสระ (สามารถนำไปแสดงที่อื่นได้)
  • <section>: กลุ่มเนื้อหาที่เกี่ยวข้อง (ต้องมีหัวข้อ)
  • <aside>: เนื้อหาเสริม (sidebar, โฆษณา)
  • <footer>: ส่วนล่างสุด
  • <figure> + <figcaption>: รูปภาพ + คำบรรยาย

Multimedia: <video controls> และ <audio controls> เล่นสื่อในตัวโดยไม่ต้องลงโปรแกรมเสริม

SVG vs Canvas: <svg> = Vector (ขยายไม่แตก, DOM-based) เหมาะทำไอคอน ส่วน <canvas> = Pixel (ขับเคลื่อนด้วย JS) เหมาะทำเกม

⚠️ ข้อควรระวัง: อย่าสับสน SVG กับ Canvas! ถ้ามีวัตถุหลายพันชิ้นในหน้าจอ SVG จะช้ามากเพราะเบราว์เซอร์ต้องติดตามทุก element ใน DOM ให้ใช้ Canvas แทนในกรณีนี้ ส่วน <main> ควรมีเพียงอันเดียวต่อหน้าเพื่อ Accessibility

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: <article> ต่างจาก <section> อย่างไร?

✅ <article> เป็นเนื้อหาที่ยืนได้ด้วยตัวเอง เอาไปแสดงที่อื่นก็เข้าใจได้ (บทความ, โพสต์, รีวิว) ส่วน <section> เป็นกลุ่มเนื้อหาที่เกี่ยวข้องกัน แต่อาจต้องพึ่ง context ของหน้า

Q2: Semantic HTML ช่วย SEO ได้อย่างไร?

✅ Google's crawler เข้าใจโครงสร้างหน้าเว็บได้ดีขึ้น รู้ว่าส่วนไหนคือ nav, header, main content ทำให้ index เนื้อหาสำคัญได้ถูกต้อง ส่งผลให้อันดับการค้นหาดีขึ้น

บทที่ 16-17 Accessibility (A11y) & SEO Technical

ARIA & Roles:

  • aria-label="ชื่อปุ่ม": ตั้งชื่อให้ element สำหรับ Screen Reader
  • aria-hidden="true": บอก Screen Reader ให้ข้าม element นี้ (ใช้กับไอคอนตกแต่ง)
  • aria-live="polite": บอกให้ Screen Reader อ่านเมื่อ content เปลี่ยน (ดี loading state)
  • role="button": ระบุบทบาทของ element (ใช้เฉพาะเมื่อจำเป็นจริงๆ)
  • Keyboard Navigation: ทุก interactive element ต้อง focus ด้วย Tab ได้
  • WCAG 2.1 Level AA: มาตรฐานสากลที่รัฐบาลและบริษัทใหญ่มักบังคับใช้

SEO Technical:

  • <title>: 50-60 ตัวอักษร — แสดงบน Browser tab และผลการค้นหา
  • meta description: 150-160 ตัวอักษร — คำอธิบายใต้ชื่อในผลการค้นหา
  • Schema JSON-LD: ข้อมูลโครงสร้างที่ฝังใน <script type="application/ld+json"> ช่วยให้ Google แสดง Rich Snippets
⚠️ ข้อควรระวัง: กฎเหล็กของ A11y — "No ARIA is better than Bad ARIA" หมายความว่า <button> ธรรมดาดีกว่า <div role="button"> เพราะ native button จัดการ keyboard, focus, ARIA ให้อัตโนมัติ ส่วน div ต้องทำเองทุกอย่าง

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ทดสอบ Accessibility ของเว็บอย่างไรได้บ้าง?

✅ (1) ลอง navigate ด้วย Tab key อย่างเดียวโดยไม่ใช้เมาส์ (2) ใช้ Lighthouse ใน Chrome DevTools (3) ติดตั้ง Screen Reader เช่น NVDA (Windows ฟรี) (4) ใช้ extension axe DevTools

Q2: Schema JSON-LD ช่วยอะไรบน Google?

✅ ทำให้ Google แสดง Rich Snippets เช่น ดาว Rating ในผลการค้นหา, FAQ ที่ขยายได้, Recipe ที่มีรูปและเวลาทำอาหาร — เพิ่ม Click-through Rate ได้มาก
