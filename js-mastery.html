บทที่ 1-7 JavaScript Fundamentals, Operators, Scope & Closures

Operators: การเปรียบเทียบใน JS มีสองระดับคือ == (ตรวจแค่ค่าข้างในโดยจะพยายามแปลงประเภทข้อมูลให้เท่ากันอัตโนมัติ) และ === (ตรวจสอบเข้มงวด ทั้งประเภทข้อมูลและค่าข้างในต้องเหมือนกันทั้งหมด)

Scope (ขอบเขตตัวแปร): การประกาศตัวแปรด้วย let และ const จะผูกอยู่ภายใต้ขอบเขตบล็อกวงเล็บปีกกา {} (Block Scope) ช่วยป้องกันไม่ให้ค่าตัวแปรหลุดไหลไปกระทบส่วนอื่น

Closures (หัวข้อเด่นที่มักถูกถามตอนสัมภาษณ์งาน): คือความสามารถของฟังก์ชันลูกในการจดจำและเข้าถึงตัวแปรที่อยู่รอบนอกตัวมันได้ แม้ว่าฟังก์ชันตัวแม่จะทำงานจบและถูกเคลียร์ทิ้งไปแล้วก็ตาม

Data Types: JavaScript มี 7 ชนิดข้อมูลพื้นฐาน ได้แก่ string, number, boolean, null, undefined, symbol, bigint และ 1 ชนิดซับซ้อนคือ object (ซึ่งรวม Array และ Function ด้วย)

⚠️ ข้อควรระวัง: หลีกเลี่ยงการใช้คำสั่ง var ในการประกาศตัวแปรยุคนี้โดยเด็ดขาด เนื่องจาก var จะไม่สนใจ Block Scope และจะทำให้เกิดปรากฏการณ์ "Hoisting" ซึ่งมักส่งผลให้โปรแกรมทำงานผิดพลาดอย่างหาจุดผิดได้ยาก

บทที่ 8-9 Objects & High-Order Array Methods

Objects & 'this': คีย์เวิร์ด this จะเปลี่ยนค่าแปรผันตาม "วิธีการเรียกใช้ฟังก์ชัน" หากเรียกในฟังก์ชันปกติจะหมายถึงออบเจกต์เจ้าของ แต่หากเรียกใช้ใน Arrow Function คีย์เวิร์ด this จะไม่มีเป็นของตัวเอง แต่จะดึงค่า this จากสภาพแวดล้อมที่ครอบอยู่ภายนอกมาใช้

High-Order Array Methods:

  • .map(): วนลูปแปลงข้อมูลและส่งอาเรย์ใหม่ออกมา (ขนาดเท่าเดิม)
  • .filter(): คัดกรองเฉพาะสมาชิกที่ผ่านเงื่อนไข
  • .reduce(): รวบรวมข้อมูลทั้งหมดออกมาเป็นค่าเดียว (เช่น ผลรวม)
  • .find(): ค้นหาสมาชิกตัวแรกที่ตรงเงื่อนไข
  • .some() / .every(): ตรวจสอบว่ามีบางตัว/ทุกตัวที่ผ่านเงื่อนไขหรือไม่
⚠️ ข้อควรระวัง: เมธอดอย่าง map และ filter จะส่งข้อมูลชุดใหม่ออกมาเสมอโดยไม่ยุ่งกับต้นฉบับ แต่ระวัง! เมธอดอย่าง .sort(), .reverse(), .push(), และ .splice() จะเข้าไปดัดแปลงอาเรย์ต้นฉบับโดยตรง (Mutation)

บทที่ 10-13 DOM, Events & Browser Storage

DOM Manipulation: การเข้าถึงและแก้ไขอิลิเมนต์หน้าเว็บ:

  • document.querySelector(): เลือก Element แรกที่ตรง CSS Selector
  • element.textContent: อ่าน/เขียนข้อความ (ไม่แปลง HTML)
  • element.innerHTML: อ่าน/เขียนโค้ด HTML (ระวัง XSS)
  • element.addEventListener('click', fn): ผูก Event Handler

Browser Storage: localStorage (บันทึกถาวร ไม่หายเมื่อปิดเบราว์เซอร์) และ sessionStorage (ลบเมื่อปิดแท็บ)

JSON Processing: JSON.stringify(obj) แปลง Object → String และ JSON.parse(str) แปลง String → Object

⚠️ ข้อควรระวัง: การดึงข้อมูลจาก localStorage ด้วย JSON.parse() หากข้อมูลเป็น null หรือรูปแบบผิด โปรแกรมจะแครชทันที ต้องครอบด้วย try...catch เสมอ นอกจากนี้ innerHTML อาจทำให้เกิดช่องโหว่ XSS ถ้ารับข้อมูลมาจากผู้ใช้โดยตรง

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: querySelector ต่างจาก querySelectorAll อย่างไร?

✅ querySelector คืน Element ตัวแรกที่ตรง (หรือ null) ส่วน querySelectorAll คืน NodeList ของทุกตัวที่ตรง

Q2: ทำไม localStorage.setItem ต้องใช้ JSON.stringify กับ Object?

✅ LocalStorage เก็บได้แค่ String เท่านั้น ถ้าใส่ Object เข้าไปโดยตรงจะถูกแปลงเป็น "[object Object]" โดยอัตโนมัติ ต้อง stringify ก่อนเสมอ

Q3: Event Bubbling คืออะไร?

✅ เมื่อ Event เกิดที่ Element ลูก มันจะ "ฟองขึ้น" ผ่านทุก parent จนถึง document ทำให้ parent ก็รับ Event ด้วยถ้ามี listener อยู่ ใช้ e.stopPropagation() เพื่อหยุด

บทที่ 14-16 Async JavaScript, Fetch API & Error Handling

Asynchronous: การทำงานแบบไม่รอช้า คำสั่งที่ต้องใช้เวลา (เช่น ดึงข้อมูลจาก Server) จะถูกแยกไปทำเบื้องหลัง ไม่ให้หน้าเว็บค้าง

Promise: วัตถุที่แทนค่าผลลัพธ์ในอนาคต มี 3 สถานะ — pending, fulfilled, rejected

Async / Await: ไวยากรณ์สมัยใหม่ที่ทำให้เขียนโค้ด async อ่านง่ายเหมือนโค้ดปกติ — ต้องใช้คู่กันเสมอ (await ต้องอยู่ใน async function)

Fetch API: ส่งคำร้องขอ HTTP ผ่าน Methods: GET (ดึงข้อมูล), POST (สร้าง), PUT/PATCH (แก้ไข), DELETE (ลบ)

⚠️ ข้อควรระวัง: fetch() จะ reject (วิ่งเข้า catch) เฉพาะตอนเครือข่ายล้มเหลวเท่านั้น ถ้า server ตอบ 404 หรือ 500 จะถือว่าสำเร็จ! ต้องเช็ก if (!response.ok) เพิ่มเติมเสมอ และต้องครอบทั้งหมดด้วย try...catch

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: Promise.all([p1, p2, p3]) ทำงานอย่างไร?

✅ รัน Promise ทุกตัวพร้อมกัน และรอจนทุกตัวสำเร็จ ถ้าตัวใดตัวหนึ่ง reject ทั้งหมดจะ reject ทันที ใช้เมื่อต้องการดึงข้อมูลหลายอย่างพร้อมกันเพื่อประหยัดเวลา

Q2: fetch คืนค่าอะไร และต้องแปลงอย่างไรถึงจะใช้ข้อมูล JSON ได้?

✅ fetch คืน Promise ของ Response object ต้องเรียก await response.json() อีกครั้งเพื่อแปลงเนื้อหาเป็น JS Object

Q3: ความแตกต่างของ try/catch กับ .catch() คืออะไร?

✅ ทั้งคู่ทำงานเหมือนกัน แต่ try/catch ใช้กับ async/await ทำให้อ่านง่ายกว่า ส่วน .catch() ใช้กับ Promise chain แบบเดิม

บทที่ 17-19 Modules, OOP & Functional Programming

บทที่ 17 — ES Modules

Modules คืออะไร: ระบบแบ่งโค้ดออกเป็นไฟล์ย่อยๆ ที่มีหน้าที่เฉพาะตัว แล้วเรียกใช้งานข้ามไฟล์ได้ ช่วยให้โค้ดจัดระเบียบง่าย ทดสอบได้ และนำกลับมาใช้ซ้ำได้

  • export: ประกาศส่งออกฟังก์ชัน, ตัวแปร, หรือ Class ออกจากไฟล์
  • export default: ส่งออกค่าหลักหนึ่งค่าต่อไฟล์ (นำเข้าโดยไม่ต้องใช้ {})
  • import { fn } from './file.js': นำเข้าแบบเจาะจงชื่อ (Named Import)
  • import * as utils from './file.js': นำเข้าทั้งหมดและตั้งชื่อกลุ่ม
⚠️ ข้อควรระวัง: ES Modules ทำงานบน type="module" script tag เท่านั้น และต้องรันผ่าน server (ไม่สามารถเปิดจาก file:// ตรงๆ ได้เพราะ CORS policy) ในโปรเจกต์จริงให้ใช้ Vite หรือ Webpack จัดการ Module Bundling ให้
💡 Pro Tip: ในโปรเจกต์ใหญ่ ให้จัดโครงสร้าง Module ตาม Feature ไม่ใช่ตาม Type เช่น โฟลเดอร์ auth/ มีทั้ง auth.js, authUtils.js, authTypes.js ไว้ด้วยกัน แทนที่จะแยกเป็นโฟลเดอร์ utils/ และ types/

บทที่ 18 — OOP (Object-Oriented Programming)

Class: แม่แบบ (Blueprint) สำหรับสร้างวัตถุ (Object) ที่มีโครงสร้างเหมือนกัน

  • constructor(): เมธอดพิเศษที่รันทันทีเมื่อสร้าง Object ด้วย new
  • extends: การสืบทอด (Inheritance) — คลาสลูกได้รับทุกอย่างจากคลาสแม่
  • super(): เรียกใช้ constructor หรือ method ของคลาสแม่
  • Encapsulation: ซ่อนข้อมูลภายใน class ด้วย #privateField
  • Polymorphism: คลาสลูกสามารถ Override method ของคลาสแม่ให้ทำงานต่างออกไปได้

บทที่ 19 — Functional Programming

Pure Function: ฟังก์ชันที่ (1) ให้ผลลัพธ์เดิมเสมอสำหรับ input เดิม และ (2) ไม่มี Side Effects (ไม่แก้ตัวแปรนอก ไม่เรียก API ไม่เขียน DOM) — ทำให้ทดสอบง่ายมาก

Immutability: ไม่แก้ไขข้อมูลเดิมโดยตรง แต่สร้างข้อมูลใหม่แทน เช่นใช้ spread operator หรือ Object.freeze()

Higher Order Function (HOF): ฟังก์ชันที่รับฟังก์ชันอื่นเป็น argument หรือคืนค่าเป็นฟังก์ชัน เช่น .map(), .filter(), .reduce()

Currying: เทคนิคแปลงฟังก์ชันที่รับหลาย argument ให้กลายเป็นฟังก์ชันหลายชั้น รับ argument ทีละตัว — ช่วยให้ reuse ได้ง่ายขึ้น

⚠️ ข้อควรระวัง: Functional Programming ไม่ใช่เรื่องของการใช้ arrow function ทุกที่ แต่คือ "วิธีคิด" — หลีกเลี่ยงการ Mutate State และแยก Side Effects ออกจาก Business Logic ให้ชัดเจน

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ฟังก์ชันนี้เป็น Pure Function ไหม? function add(a,b){ return a+b; }

✅ ใช่ — ให้ผลเดิมเสมอสำหรับ input เดิม และไม่มี Side Effects

Q2: export default ต่างจาก export แบบปกติอย่างไร?

✅ export default มีได้แค่อันเดียวต่อไฟล์ ตอนนำเข้าไม่ต้องใช้ {} และตั้งชื่อใหม่ได้เลย ส่วน Named Export มีได้หลายอัน แต่ต้องใช้ {} และชื่อต้องตรง

บทที่ 20-22 Event Loop & Memory Management (Senior Spec)

กลไกเชิงลึกของภาษา JavaScript (Single-Threaded Runtime):

เนื่องจากเครื่องยนต์ JS ประมวลผลโค้ดได้ทีละคำสั่ง มันจึงใช้ระบบสลับงานอย่างแยบยล:

  • Call Stack: ประมวลผลโค้ดทั่วไปแบบเข้าทีหลังออกก่อน (LIFO)
  • Microtask Queue: แถวคิวพิเศษระดับสูงสุดของ .then() และ async/await — รันก่อน Macrotask เสมอ
  • Macrotask / Callback Queue: แถวงานทั่วไป เช่น setTimeout, setInterval, DOM Events
  • กฎเหล็ก: Event Loop จะดึงงานจาก Microtask Queue ให้หมดก่อน แล้วค่อยดึง Macrotask มาทำทีละอัน

Memory Layout: ตัวแปร primitive อยู่ใน Stack (เร็ว) ส่วน Object/Array อยู่ใน Heap (ยืดหยุ่น) พร้อมระบบ Garbage Collection ที่คอยเก็บกวาดหน่วยความจำที่ไม่มีใครอ้างอิงอีกต่อไป

⚠️ ข้อควรระวังระดับสูง: การผูก Event Listener ไว้กับ window หรือตั้ง setInterval ทิ้งไว้โดยไม่เคยล้างด้วย removeEventListener หรือ clearInterval จะทำให้เกิด Memory Leak — Object จะค้างอยู่ใน Heap ตลอดไปจนระบบช้าลง

🧠 Quiz — คลิกที่คำถามเพื่อดูคำตอบ

Q1: ถ้า Call Stack ไม่ว่าง Event Loop จะทำงานไหม?

✅ ไม่ — Event Loop จะรอจนกว่า Call Stack ว่างเปล่าก่อน แล้วค่อยดึงงานจาก Queue มา ทำให้ถ้า Synchronous code ทำงานนานมากหน้าเว็บจะค้าง (ไม่ตอบสนอง)

Q2: ทำไม async/await ถึงใช้ Microtask Queue?

✅ เพราะ await ถูก compile เป็น .then() โดย JS Engine ทำให้ code หลัง await วิ่งใน Microtask Queue ซึ่งมี priority สูงกว่า setTimeout

  • Package Manager: แนะนำ pnpm แทน npm เพราะใช้ Hard Links ประหยัดพื้นที่ฮาร์ดดิสก์และเร็วกว่า
  • TypeScript: JavaScript + ระบบ Type ช่วยจับข้อผิดพลาดตั้งแต่เขียนโค้ด ไม่ต้องรอให้ runtime crash
  • Next.js Render Modes: SSR (เรนเดอร์สดจากเซิร์ฟเวอร์ทุกครั้ง), SSG (คอมไพล์ตั้งแต่ build time โหลดเร็วสุด), ISR (อัปเดตหน้า SSG ตามเวลาที่กำหนด)
  • Docker: ห่อโปรแกรมพร้อม environment ไว้ใน Container ทำให้ "ทำงานบนเครื่องฉันได้" กลายเป็น "ทำงานได้ทุกเครื่อง"
💡 แนวทางการเรียนต่อ: หลังจากเชี่ยวชาญ JS แล้ว เส้นทางที่แนะนำคือ TypeScript → React → Next.js → Node.js/Express → Database (PostgreSQL หรือ MongoDB) → Docker → Cloud (AWS/GCP/Vercel) ในแต่ละขั้นให้สร้าง project จริงเก็บไว้ใน GitHub เพื่อใช้ในการสมัครงาน
